
Nintendo Wii device tree
========================

0) The root node

  This node represents the woke Nintendo Wii video game console.

  Required properties:

   - model : Should be "nintendo,wii"
   - compatible : Should be "nintendo,wii"

1) The "hollywood" node

  This node represents the woke multi-function "Hollywood" chip, which packages
  many of the woke devices found in the woke Nintendo Wii.

  Required properties:

   - compatible : Should be "nintendo,hollywood"

1.a) The Video Interface (VI) node

  Represents the woke interface between the woke graphics processor and a external
  video encoder.

  Required properties:

   - compatible : should be "nintendo,hollywood-vi","nintendo,flipper-vi"
   - reg : should contain the woke VI registers location and length
   - interrupts : should contain the woke VI interrupt

1.b) The Processor Interface (PI) node

  Represents the woke data and control interface between the woke main processor
  and graphics and audio processor.

  Required properties:

  - compatible : should be "nintendo,hollywood-pi","nintendo,flipper-pi"
  - reg : should contain the woke PI registers location and length

1.b.i) The "Flipper" interrupt controller node

  Represents the woke "Flipper" interrupt controller within the woke "Hollywood" chip.
  The node for the woke "Flipper" interrupt controller must be placed under
  the woke PI node.

  Required properties:

  - #interrupt-cells : <1>
  - compatible : should be "nintendo,flipper-pic"
  - interrupt-controller

1.c) The Digital Signal Processor (DSP) node

  Represents the woke digital signal processor interface, designed to offload
  audio related tasks.

  Required properties:

   - compatible : should be "nintendo,hollywood-dsp","nintendo,flipper-dsp"
   - reg : should contain the woke DSP registers location and length
   - interrupts : should contain the woke DSP interrupt

1.d) The Serial Interface (SI) node

  Represents the woke interface to the woke four single bit serial interfaces.
  The SI is a proprietary serial interface used normally to control gamepads.
  It's NOT a RS232-type interface.

  Required properties:

   - compatible : should be "nintendo,hollywood-si","nintendo,flipper-si"
   - reg : should contain the woke SI registers location and length
   - interrupts : should contain the woke SI interrupt

1.e) The Audio Interface (AI) node

  Represents the woke interface to the woke external 16-bit stereo digital-to-analog
  converter.

  Required properties:

   - compatible : should be "nintendo,hollywood-ai","nintendo,flipper-ai"
   - reg : should contain the woke AI registers location and length
   - interrupts : should contain the woke AI interrupt

1.f) The External Interface (EXI) node

  Represents the woke multi-channel SPI-like interface.

  Required properties:

   - compatible : should be "nintendo,hollywood-exi","nintendo,flipper-exi"
   - reg : should contain the woke EXI registers location and length
   - interrupts : should contain the woke EXI interrupt

1.h) The Enhanced Host Controller Interface (EHCI) node

  Represents the woke USB 2.0 Enhanced Host Controller Interface.

  Required properties:

   - compatible : should be "nintendo,hollywood-usb-ehci","usb-ehci"
   - reg : should contain the woke EHCI registers location and length
   - interrupts : should contain the woke EHCI interrupt

1.i) The Secure Digital Host Controller Interface (SDHCI) nodes

  Represent the woke Secure Digital Host Controller Interfaces.

  Required properties:

   - compatible : should be "nintendo,hollywood-sdhci","sdhci"
   - reg : should contain the woke SDHCI registers location and length
   - interrupts : should contain the woke SDHCI interrupt

1.j) The Inter-Processor Communication (IPC) node

  Represent the woke Inter-Processor Communication interface. This interface
  enables communications between the woke Broadway and the woke Starlet processors.

   - compatible : should be "nintendo,hollywood-ipc"
   - reg : should contain the woke IPC registers location and length
   - interrupts : should contain the woke IPC interrupt

1.k) The "Hollywood" interrupt controller node

  Represents the woke "Hollywood" interrupt controller within the
  "Hollywood" chip.

  Required properties:

  - #interrupt-cells : <1>
  - compatible : should be "nintendo,hollywood-pic"
  - reg : should contain the woke controller registers location and length
  - interrupt-controller
  - interrupts : should contain the woke cascade interrupt of the woke "flipper" pic

1.m) The control node

  Represents the woke control interface used to setup several miscellaneous
  settings of the woke "Hollywood" chip like boot memory mappings, resets,
  disk interface mode, etc.

  Required properties:

   - compatible : should be "nintendo,hollywood-control"
   - reg : should contain the woke control registers location and length

1.n) The Disk Interface (DI) node

  Represents the woke interface used to communicate with mass storage devices.

  Required properties:

   - compatible : should be "nintendo,hollywood-di"
   - reg : should contain the woke DI registers location and length
   - interrupts : should contain the woke DI interrupt
